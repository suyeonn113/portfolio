다이소몰 포트폴리오 정리
작성일: 2026-07-07
상태: 제출용 정리 확정 완료


daisomall 완성본 요약

포지셔닝
- "다이소몰 전체 클론"이 아니라 "상품 탐색 경험 중심 커머스 UI 리디자인"으로 설명한다.
- 팀 프로젝트 결과물을 제출용으로 다시 정리하면서 반응형, 접근성, 정보 위계, 검색 UX를 보완한 프로젝트로 잡는다.
- 상품 상세는 Fragfarm에서도 보여줄 수 있으므로, daisomall에서는 상세 페이지보다 홈+검색 탐색 UX를 강점으로 삼는다.

대표 사용자 흐름
- 홈에서 프로모션, 배송 서비스, 카테고리, 기획전, 랭킹, 추천 상품을 훑는다.
- 검색 페이지에서 키워드를 입력하거나 추천 키워드를 선택한다.
- 검색 결과에서 배송 옵션과 정렬 기준으로 상품을 좁힌다.
- 상품 카드에서 가격, 평점, 배송 가능 여부를 빠르게 비교한다.

면접 핵심 메시지
- "상품 상세까지 넓게 만들기보다, 커머스에서 사용 빈도가 높은 홈 탐색과 검색 결과 탐색 흐름을 집중적으로 다듬었습니다."
- "팀 프로젝트 당시 React 숙련도가 높지 않아 페이지 수는 많지 않았지만, 이후 제출용으로 정리하면서 반응형, 접근성, 정보 위계, 검색 UX를 보완했습니다."
- "접근성은 체크리스트처럼 넣은 것이 아니라, 실제 키보드 사용자가 홈과 검색 흐름을 따라갈 수 있는지 기준으로 정리했습니다."


daisomall 기술 포인트

구조
- React Router로 홈/검색 중심 라우팅을 구성했다.
- 상품 데이터를 분리하고 검색/필터/정렬 상태를 React state와 hook으로 관리했다.
- MainLayout을 기준으로 Header, MainCategoryNav, BottomTabBar, Footer를 공통 레이아웃으로 구성했다.
- ProductCard, ProductCardList, SectionHeader, BadgeButton 등 반복 UI를 공통 컴포넌트로 분리했다.
- 홈 섹션은 배송, 카테고리, 기획전, 랭킹, 추천 상품처럼 역할별 섹션 컴포넌트로 나눴다.

스타일
- SCSS 토큰과 breakpoint를 사용해 여백, 색상, radius, focus ring 기준을 맞췄다.
- 상품 목록은 모바일에서 스크롤/리스트형, 데스크톱에서 더 넓은 그리드형으로 보이게 조정했다.
- focus-visible 두께와 radius를 토큰 기준으로 통일했다.
- 큰 카드, 배너, 이미지 영역은 다이소몰의 실용적인 브랜드 인상에 맞춰 거의 네모에 가까운 radius로 정리했다.
- 버튼, 칩, 검색창, 하단 앱형 컨트롤은 모바일 조작감이 떨어지지 않도록 부드러운 radius를 유지했다.
- 홈 섹션 간격은 실제 화면 기준으로 다시 점검해 공통 섹션 padding, 카테고리-추천상품 간격, 배송-푸터 간격을 조정했다.
- 이미지형 배너와 기획전처럼 이미지가 강한 영역은 포커스링이 묻히지 않도록 내부 오버레이 방식으로 처리했다.

검색 UX
- 검색 입력, 추천 키워드, 검색 결과, 배송 필터, 정렬 버튼 흐름을 구성했다.
- 검색 결과 요약은 현재 검색 상태를 문장으로 보여주고, 필터/정렬 선택 상태를 시각적으로 구분했다.
- 상품 카드에서 가격, 상품명, 평점, 배송 뱃지를 빠르게 비교할 수 있게 했다.


daisomall 접근성 포인트

키보드 흐름
- 스킵 메뉴를 추가해 반복되는 헤더와 내비게이션을 건너뛰고 본문/주요 메뉴로 이동할 수 있게 했다.
- 이동하지 않는 메뉴는 링크가 아니라 버튼으로 바꿔 link/button 역할을 구분했다.
- 랭킹과 오늘의 추천상품 카드도 키보드 포커스가 가도록 했다.
- 랭킹/기획전 카테고리는 탭 리스트 패턴으로 정리해 Tab은 콘텐츠 영역으로 이동하고, 카테고리 간 이동은 방향키로 처리했다.
- 카테고리 드로어는 대분류 선택 후 서브 카테고리로 포커스가 들어가고, 서브 안에서는 방향키로 이동하며 메인 카테고리로 돌아갈 수 있게 했다.

스크린 리더 정보
- 검색 input, 검색 결과, 필터/정렬, 상품 목록에 필요한 aria 정보를 보완했다.
- 필터와 정렬은 선택 상태가 중요하므로 aria-pressed로 현재 상태를 전달했다.
- 랭킹/기획전 탭은 role, aria-selected, aria-controls를 사용해 선택 구조를 전달했다.
- 홍보 배너와 프로모션 이미지처럼 같은 카드 안에 제목 텍스트가 있는 이미지는 빈 alt로 처리해 중복 낭독을 줄였다.
- 상품 카드처럼 카드 자체에 상품명/가격 라벨을 제공하는 경우 이미지 alt 중복을 피했다.


daisomall 컴포넌트 설명 포인트

ProductCard / ProductCardList
- 상품 카드의 이미지, 라벨, 정보, 배송 뱃지 역할을 분리했다.
- 일반 상품 목록과 랭킹 상품 목록을 같은 컴포넌트 계열에서 처리하되, 랭킹일 때 순위와 키보드 포커스 라벨을 추가했다.

SearchPage / SearchResultsView
- 검색어, 필터, 정렬 상태를 관리하고 결과 화면에 전달한다.
- 검색 결과 요약, 필터/정렬 버튼, 상품 목록을 묶어 검색 경험을 구성한다.

MainLayout
- 공통 레이아웃, 스킵 메뉴, Header/Footer/BottomTabBar 구조를 담당한다.
- 홈과 검색이 같은 서비스 안의 화면처럼 느껴지도록 기본 UI 틀을 유지한다.

CategoryDrawer
- 일반 탭 패턴을 억지로 쓰지 않고, 왼쪽 대분류와 오른쪽 여러 서브 섹션이 이어지는 구조에 맞춰 사이드 내비게이션으로 정리했다.
- 대분류 선택, 서브 카테고리 진입, 방향키 이동, 메인 카테고리 복귀 흐름을 키보드로 사용할 수 있게 했다.

RankingTabList / PromotionTabList
- 모든 탭을 Tab으로 지나가게 하지 않고, 탭 그룹 안에서는 방향키로 이동하고 Enter/Space로 선택하도록 정리했다.
- 선택된 탭과 연결된 콘텐츠 영역을 aria-controls로 연결했다.

Footer
- 정보량이 많아 세분화된 구조를 유지했다.
- 고객센터, 메뉴, 앱 다운로드, 회사 정보, 정책/인증 정보를 역할별로 나눠 가독성을 확보했다.

에어서울 포트폴리오 정리
작성일: 2026-07-08
상태: 제출용 정리 확정 완료


airseoul 완성본 요약

포지셔닝
- "항공사 전체 서비스 경험 리디자인"으로 설명한다.
- 단순 랜딩 페이지가 아니라 항공권 검색/예약, 예약 조회, 체크인, 스케줄, 변경/환불, 여행 정보, 이벤트, 브랜드 소개까지 라우팅으로 구성한 서비스형 포트폴리오로 잡는다.
- 모든 항공사 메뉴를 끝까지 구현한 클론이 아니라, 포트폴리오에서 보여줄 핵심 흐름을 중심으로 구현한 리디자인 프로젝트라고 설명한다.
- 다이소몰이 홈+검색 중심 커머스라면, airseoul은 항공권 예약 탐색과 복잡한 정보 구조를 보여주는 프로젝트로 구분한다.

페이지 볼륨
- 제출용 페이지 수는 부족하지 않다.
- 실제 라우팅 화면은 홈, 항공권 예매, 항공편 검색 결과, 예약 조회, 체크인, 출도착/스케줄, 예약 변경/환불, 브랜드 스토리, 이벤트 목록, 이벤트 상세, 공항 서비스, 수하물, 좌석으로 구성되어 있다.
- 메뉴에 일부 placeholder 항목이 있으므로 면접에서는 "항공사 사이트 전체 메뉴 중 핵심 예약/여행 정보 흐름을 구현 범위로 정했다"고 말하는 것이 좋다.

대표 사용자 흐름
- 홈에서 여행지 히어로, 항공권 검색, 빠른 메뉴, 에어서울 소개 정보를 확인한다.
- 항공권 검색 조건을 입력해 URL query 기반 검색 결과 화면으로 이동한다.
- 검색 결과에서 날짜별 운임을 확인하고, 가는편/오는편 항공편과 운임을 선택한다.
- 여정 요약 aside에서 선택한 항공편, 탑승객, 총액을 비교한다.
- 예약 조회, 체크인, 출도착/스케줄 화면에서 항공사 서비스의 후속 업무 흐름을 확인한다.
- 좌석, 수하물, 공항 서비스 페이지에서 정보형 페이지의 탭/표/안내 카드 구조를 확인한다.

면접 핵심 메시지
- "항공권 검색 조건 입력부터 검색 결과 비교, 운임 선택, 여정 요약까지 이어지는 예약 탐색 흐름을 구현했습니다."
- "단순 메인 페이지가 아니라 예약, 체크인, 스케줄, 좌석, 수하물, 이벤트, 브랜드 소개 등 항공사 사이트의 핵심 정보 구조를 라우팅으로 나눴습니다."
- "검색 조건을 URLSearchParams로 전달하고, 결과 화면에서 조건을 다시 반영해 서비스 흐름처럼 동작하게 했습니다."
- "탭, 날짜 선택, 운임 선택처럼 사용자가 반복해서 조작하는 UI는 키보드 접근성과 focus-visible 표시를 함께 정리했습니다."
- "SCSS foundation, token, page/component partial 구조를 사용해 색상, 여백, hover, focus ring 기준을 통일했습니다."


airseoul 기술 포인트

구조
- React Router로 주요 서비스 페이지를 구성했다.
- MainLayout을 기준으로 Header, Footer, 로그인 패널, 접근성 빠른 메뉴를 공통 레이아웃으로 묶었다.
- 항공권 예매 패널은 bookingPanelData로 데이터성 설정을 분리하고, 컴포넌트는 화면 상태와 이벤트 처리에 집중하도록 정리했다.
- 검색 결과 화면은 URLSearchParams에서 검색 조건을 읽고, 조건에 맞는 항공편/운임 데이터를 계산해 화면에 보여준다.
- 체크인과 스케줄 SCSS는 500줄을 넘지 않도록 상세 안내/결과 영역을 partial로 분리했다.

예약/검색 로직
- createFixedRoundTripSearchParams, createOneWaySearchParams로 검색 query 생성 흐름을 분리했다.
- FlightSearchResults에서 출발지, 도착지, 날짜, 탑승객 수를 읽어 가는편/오는편 결과를 구성한다.
- DateFareBar에서 날짜별 운임을 보여주고, 키보드 방향키로 날짜를 이동할 수 있게 했다.
- FlightOptionRow에서 항공편별 운임을 선택하고, BookingSummaryAside에서 선택 결과를 요약한다.

스타일 시스템
- SCSS foundation에 token, breakpoint, mixin을 두고 공통 기준을 관리했다.
- focus ring 두께는 전역 `--focus-ring-width` 기준으로 통일했다.
- 잘리는 탭 포커스링은 `clipped-focus-ring` 믹스로 내부 box-shadow 방식으로 정리했다.
- hover는 선택 상태와 구분되도록 정리했다. 활성/선택은 브랜드 민트, 단순 hover는 중립 쿨그레이 계열로 구분한다.
- 페이지 상단, 제목-탭-본문 간격, 카드 내부 padding을 실제 화면 기준으로 조정했다.
- 브랜드 스토리의 가치 섹션은 네모 카드 대신 원형 노드형 삼각 구조로 정리해 섹션의 시각적 밀도를 높였다.


airseoul 접근성 포인트

키보드 흐름
- 접근성 빠른 메뉴를 추가해 본문, 주요 메뉴, 푸터로 바로 이동할 수 있게 했다.
- 주요 탭 UI는 roving tabindex 패턴으로 정리했다. Tab은 탭 그룹에 한 번 진입하고, 탭 내부 이동은 방향키/Home/End로 처리한다.
- 적용 범위는 수하물, 좌석, 공항 서비스, 체크인, 스케줄, 로그인 패널, 운임 날짜 선택 등이다.
- 항공편 날짜 선택은 Tab으로 모든 날짜를 지나가지 않도록 하고, 선택 날짜 기준으로 방향키 이동이 가능하게 했다.
- 운임 선택 카드는 실제 포커스가 input에 있어도 카드 전체에 focus-within 링이 보이게 했다.

스크린 리더 정보
- 탭과 패널은 role, aria-selected, aria-controls, aria-labelledby로 연결했다.
- 예약 단계는 aria-current="step"으로 현재 단계를 전달한다.
- 선택 상태가 필요한 버튼에는 aria-pressed 또는 aria-selected를 사용했다.
- 로그인 패널은 dialog, aria-modal, aria-labelledby를 사용해 별도 패널 구조로 전달한다.
- 장식용 아이콘은 aria-hidden 처리하고, 의미가 필요한 버튼은 aria-label을 제공했다.

포커스/시각 피드백
- 홈 예약 탭, 상세 페이지 탭, 로그인 탭, 정보형 탭의 focus ring 두께와 위치를 통일했다.
- 컨테이너 overflow 때문에 외곽 outline이 잘리는 곳은 내부 focus ring으로 변경했다.
- 항공편 운임 가격 카드처럼 포커스가 잘 보이지 않던 영역은 내부 focus-within 처리로 보완했다.
- 좌석 배치도 이미지는 테두리와 이미지가 붙지 않도록 내부 여백을 추가했다.

- 청소년 프로그램 아래에는 참여활동, 수련활동, 지역연계활동, 학교연계활동, 진로체험활동, 미래융합활동, 국제교류활동, 장애청소년활동, 특성화활동, 동그라미학교, 방과후아카데미, 청소년문화공간을 배치했다.
- 프로그램 신청은 시각적으로 구분되는 독립 메뉴이며 청소년 프로그램 신청, 평생교육 프로그램 신청, 신청내역 확인으로 구성했다.
- PC 메가메뉴, 모바일 메뉴, 좌측 서브메뉴에서 같은 메뉴 위계와 명칭을 유지했다.

청소년 프로그램 신청
- programs.php는 접수 가능한 프로그램만 보여주고 상태, 연령, 분야, 검색어 필터를 제공한다.
- 접수 상태 용어는 사용자에게 익숙한 "접수중"으로 통일했다.
- 카드에서 상세 페이지, 신청 폼, 신청 확인 흐름으로 이동한다.
- 접수 종료 프로그램은 신청 목록과 상세 신청 흐름에서 제외했다.
- 실제 접수 규모에 맞춰 데스크톱 4열 카드 레이아웃을 사용한다.

평생교육 신청
- lifelong-education-apply.php에서 종목, 강습반, 강습명을 기준으로 접수 가능한 강좌를 탐색한다.
- 강좌명, 장소, 종목, 강사, 요일·시간, 대상, 수강료, 정원, 접수 상태와 신청 버튼을 표로 제공한다.
- 강좌별 접수 인원을 진행 바로 시각화하고 프로젝트의 짙은 남색을 사용해 좌측 패널과 색상 체계를 맞췄다.
- 신청 폼과 actions/lifelong_application_create.php를 연결해 평생교육 신청 데이터를 저장한다.

신청내역과 데이터
- 청소년 프로그램과 평생교육 프로그램을 위아래 별도 표로 나눠 한 화면에서 확인하게 했다.
- 두 표는 프로그램명, 신청자, 연락처, 신청일, 접수상태, 바로가기 열 너비를 같은 그리드로 통일했다.
- program_type 컬럼으로 청소년 프로그램과 평생교육 신청을 구분한다.
- 신청 상세, 수정, 삭제 처리 흐름을 유지했다.

홈 화면
- 히어로는 하나의 이미지를 PC와 모바일에서 공용으로 사용하는 조건에 맞춰 서로 다른 콘셉트의 배너 5종으로 구성했다.
- 배너 이동 버튼은 이미지의 핵심 텍스트를 가리지 않도록 바깥쪽으로 배치하고 컨트롤러는 배너 기준 오른쪽에 정리했다.
- 접수중 프로그램은 6개 기준의 전시형 자동 슬라이드로 구성하고 좌우 한 장 이동 기능을 제공한다.
- 프로그램 포스터는 교육, 시민참여, 문화예술 등 주제별로 색상, 사진, 일러스트, 타이포 콘셉트를 다르게 제작했다.
- 센터 소식과 일정은 공지사항, 서울시정, 보도자료, 달력, 센터 일정을 하나의 큰 섹션 안에서 균형 있게 배치했다.
- 일정 수가 변해도 카드 위치와 양쪽 열 높이가 흔들리지 않으며 일정이 많을 때는 센터 일정 카드 안에서만 스크롤된다.
- 공지 제목은 한 줄 말줄임으로 처리해 콘텐츠 길이에 따라 레이아웃이 덜컹거리지 않게 했다.
- 활동사진은 데스크톱 콜라주와 작은 화면 3열 구조를 사용하고, 인생네컷 앨범 느낌의 배경으로 섹션을 구분했다.
- SNS는 대표 영상, 채널 탭, 게시물 그리드와 유관기관 보기로 구성하고 이미지 높이와 하단 정렬을 통일했다.

디자인 시스템
- 핵심 색상은 주황색과 짙은 남색이며, 주황색은 활성·행동·강조에 제한적으로 사용한다.
- 좌측 서브메뉴는 짙은 남색 제목 영역, 주황색 포인트 라인, 중립색 활성 배경으로 정리했다.
- 본문과 읽는 텍스트는 과도한 bold를 피하고 제목과 데이터 강조에만 높은 굵기를 사용한다.
- 카드 radius, 회색 테두리, 구분선, 버튼 높이와 focus ring을 공통 기준으로 맞췄다.
- 같은 카드 스타일을 반복하기보다 정보 성격에 따라 텍스트 목록, 단계형 절차, 표, 아이콘 목록을 선택해 단조로움을 줄였다.
- 넓은 화면에서는 콘텐츠 최대 폭과 타이포 크기를 확장하고, 기존 폭 이하에서는 원래 크기 체계를 유지하도록 반응형으로 조정했다.

- hover와 focus를 구분하되, 선택 상태와 단순 hover가 혼동되지 않도록 색상과 선을 정리했다.

스크린 리더 정보
- 탭과 패널은 role, aria-selected, aria-controls, aria-labelledby로 연결했다.
- 아이콘만 있는 버튼에는 aria-label을 제공하고 장식 이미지는 aria-hidden 또는 빈 대체 텍스트로 처리했다.
- 검색 결과 수, 프로그램 상태, 현재 메뉴처럼 시각적으로 강조되는 정보도 텍스트로 전달한다.

- 단순 상품 랜딩 페이지가 아니라 상품 탐색, 검색, 상세 옵션 선택, 장바구니, 위시리스트, 주문, 회원, 리뷰와 문의까지 이어지는 모바일 쇼핑몰 프로젝트로 잡는다.

fragfarm-mobile 기술 포인트

구조
- PHP 페이지와 공통 include 구조로 구성했다.
- includes/header.php, footer.php, head.php로 공통 레이아웃을 관리하고 product-card.php 등 반복 UI를 컴포넌트화했다.
- 상품 원본 데이터는 includes/data/products.php에 두고 목록, 검색, 상품 피드백은 includes/services의 함수로 분리했다.
- 페이지별 CSS, 공통 component CSS, core token/typography/color 파일을 나눠 스타일 역할을 구분했다.
- JavaScript는 헤더, 홈 히어로, 상품 목록, 상품 상세, 주문 흐름, 인증 데모, 접근성처럼 기능별 파일로 분리했다.

상품 목록/검색
- normalizeProductListState에서 category, sort, page, view query를 정규화한다.
- applyProductListState에서 필터, 품절 후순위 정렬, 선택 정렬과 페이지네이션을 순서대로 계산한다.
- 상품은 한 페이지에 10개씩 노출하고 현재 상태를 URL query로 유지한다.
- 상품 보기 방식은 1열/2열 링크로 제공해 URL 공유와 새로고침 후에도 상태가 유지된다.
- buildProductSearchHaystack에서 상품명, 카테고리, 컬러, 설명과 상태를 하나의 검색 문자열로 구성한다.
- 카테고리 alias map을 사용해 pants와 바지, skirt와 치마, top과 상의처럼 영문/한글 검색어를 함께 지원한다.

상품 카드/브랜드 UI
- 상품 카드의 이미지, 이름/가격, 리뷰, 장바구니/위시 액션을 역할별 영역으로 나눴다.
- 상품 정보는 왼쪽 정렬하고 리뷰와 액션은 카드 하단 양쪽에 배치해 모바일에서 정보와 기능을 빠르게 구분하도록 했다.
- NEW와 SALE은 장식 서체를 사용한 작은 종이 라벨 형태, SOLD OUT은 읽기 쉬운 기본 서체로 구분했다.
- 홈 리뷰 섹션에는 THE FRAGFARM ARCHIVE와 참여 문구를 추가해 사진 나열이 아니라 브랜드 기록처럼 보이도록 했다.
- 세일 화살표에는 짧은 반복 모션을, 리뷰 사진에는 마지막 뒤에 첫 사진이 연결되는 느린 무한 흐름을 적용했다.
- 자동 리뷰 흐름은 마우스와 키보드 포커스 시 멈추고 prefers-reduced-motion 환경에서는 실행하지 않는다.

장바구니/위시리스트/주문
- shop-storage.js에서 장바구니와 위시리스트 데모 상태를 일관된 데이터 형태로 정규화한다.
- 장바구니에서 수량 변경, 삭제, 선택 상품 합계, 할인과 배송비를 다시 계산한다.
- 카드의 장바구니/위시 버튼과 장바구니/위시 페이지가 같은 저장 데이터를 사용하도록 연결했다.
- 주문서, 주문 완료, 주문 목록과 상세 화면으로 이어지는 데모 주문 흐름을 구성했다.
- 실제 PG 결제나 외부 간편결제는 구현 범위에서 제외하고 준비 중 상태로 명확히 구분했다.

회원/게시판
- PHP session을 기준으로 로그인 상태와 마이페이지 분기를 구성했다.
- 회원가입, 로그인, 아이디/비밀번호 찾기, 회원 정보와 배송지 관리 화면을 연결했다.
- 상품 리뷰와 문의는 서비스 함수와 action PHP를 통해 작성, 수정, 삭제 구조를 나눴다.
- 내가 작성한 리뷰, 댓글과 문의를 마이페이지에서 다시 확인할 수 있도록 연결했다.
- DB 연결이 없는 환경에서도 포트폴리오 화면을 확인할 수 있도록 데모 데이터와 로컬 상태를 함께 사용한다.

스타일 시스템
- core/colors.css와 tokens.css에서 색상, 간격, 글자 크기, radius와 컨트롤 높이를 변수로 관리한다.
- EditorialToday 장식 서체와 기본 본문 서체를 역할에 따라 구분해 빈티지한 인상과 가독성을 함께 유지했다.
- 모바일 shell을 기준으로 페이지 너비와 공통 여백을 유지하고 상품 목록은 1열/2열 상태에 맞춰 전환한다.
- 따뜻한 아이보리, 와인, 올리브 계열을 리뷰 캡션과 상품 라벨에 제한적으로 사용해 사진보다 UI가 과하게 튀지 않게 했다.
- focus-visible, 비활성, 선택, 품절 상태가 색상만으로 전달되지 않도록 텍스트와 속성을 함께 사용했다.


fragfarm-mobile 접근성 포인트

키보드 흐름
- 메뉴와 본문 바로가기 링크를 페이지의 첫 탐색 항목으로 제공했다.
- 상품 카테고리 선택 후 새 페이지의 스킵 메뉴부터 다시 지나가지 않도록 상품 목록 제어 영역으로 포커스를 복원한다.
- 리뷰 사진처럼 가로로 이어지는 목록은 좌우 방향키 탐색과 스크롤을 지원한다.
- 상품 카드에서는 상품 링크, 장바구니, 위시리스트, 다음 상품 순서로 포커스가 이동한다.
- 품절 상품은 사용할 수 없는 장바구니 버튼을 제거하고 위시리스트 버튼만 자연스럽게 이어지게 했다.

다이얼로그/팝업
- 메뉴, 장바구니 옵션과 팝업성 UI에서 열기 버튼, dialog, 닫기 버튼의 관계를 명확히 했다.
- 다이얼로그가 열리면 내부로 포커스를 이동하고 Tab 순환을 제한한다.
- 닫은 뒤에는 사용자가 실행했던 버튼으로 포커스를 돌려보내 흐름이 끊기지 않게 했다.
- Escape와 배경 클릭 등 닫기 동작을 제공하되 키보드 사용자가 닫기 버튼을 찾을 수 있게 했다.

스크린 리더 정보
- 장식용 아이콘과 중복되는 이미지는 빈 alt 또는 aria-hidden으로 처리했다.
- 버튼에는 장바구니 추가, 위시리스트 추가/제거, 수량 변경과 삭제처럼 동작이 드러나는 이름을 제공했다.
- 평점과 리뷰 개수는 aria-label로 의미가 연결되도록 했다.
- 선택 상태는 aria-current, aria-pressed, 체크박스 상태로 전달한다.
- 토스트와 상태 변화는 role=status와 aria-live 영역을 사용해 화면 변화가 낭독되도록 했다.

모션 접근성
- 히어로 자동 전환과 리뷰 자동 흐름은 prefers-reduced-motion에서 중지한다.
- 리뷰 자동 흐름을 위해 만든 복제 카드는 aria-hidden과 tabindex=-1로 접근성 트리와 탭 순서에서 제외했다.
- 리뷰에 마우스를 올리거나 내부 링크에 키보드 포커스가 들어오면 자동 이동을 멈춰 조작을 방해하지 않게 했다.
- CSS hover/transition도 동작 줄이기 환경에서 불필요한 움직임이 남지 않도록 분기했다.

product-card.php / card.css

product-search.php

- 데이터의 영문 카테고리를 바꾸지 않고 alias map으로 한글 검색을 확장한 구조다.

shop-storage.js
- 상품 데이터를 정규화하고 장바구니, 위시리스트, 주문 데모 상태를 관리한다.
- 수량과 선택 상태가 바뀔 때 합계 금액과 헤더 개수를 함께 갱신하는 흐름을 설명할 수 있다.
- 실제 백엔드 결제 대신 포트폴리오에서 상호작용을 확인할 수 있도록 구성한 범위를 분명히 말한다.

product-feedback.php / actions
- 리뷰, 리뷰 댓글과 문의의 조회/작성/수정/삭제 흐름을 분리한 서비스 계층이다.
- 작성자 확인과 로그인 상태에 따른 동작 차이를 설명하기 좋다.
- UI만 있는 게시판이 아니라 사용자가 남긴 콘텐츠를 다시 관리하는 흐름까지 연결한 포인트다.

accessibility.js
- 스킵 링크의 실제 포커스 이동, 다이얼로그 focus trap, 가로 목록 방향키 탐색을 공통 처리한다.
- 리뷰 루프용 복제 링크를 키보드 탐색에서 제외해 시각 효과가 접근성 순서를 망가뜨리지 않게 했다.

home-hero.js / home-review-flow.js
- 홈 히어로의 자동 전환, 점 선택, 스와이프와 헤더 테마 전환을 처리한다.
- 리뷰 흐름은 원본 목록을 복제해 마지막과 첫 사진을 연결하고 scrollLeft를 순환시킨다.
- 문서 비활성, 사용자 포커스와 동작 줄이기 설정에 따라 자동 실행을 제어한다.

- product-search.php: 영문/한글 유사어 검색 구조.
- product-detail.php, product-detail.js: 옵션, 장바구니, 위시리스트와 상세 상호작용.
- shop-storage.js: 장바구니/위시/주문 데모 상태와 금액 계산.
- product-feedback.php와 actions: 리뷰/댓글/문의 CRUD 흐름.
- accessibility.js: 스킵 링크, 포커스 trap과 가로 목록 키보드 탐색.
- home-hero.js, home-review-flow.js: 자동 모션과 reduced-motion 대응.
- core tokens/colors/typography: 공통 디자인 기준.

포트폴리오 설명 시 주의할 점
- 실제 PG 결제, 소셜 로그인과 외부 간편결제를 구현했다고 말하지 않는다.
- DB와 session 기반 구조가 있지만 로컬 데모 상태를 함께 사용하므로 완전한 운영 쇼핑몰이라고 설명하지 않는다.
- 브랜드 전체 사이트 복제보다 모바일 구매 흐름과 접근성을 중심으로 재구성한 프로젝트라고 말한다.
- 상품/브랜드 이미지의 원저작권과 포트폴리오 비상업 목적을 구분해 설명한다.
- 준비 중 기능은 미완성이라고 숨기기보다 핵심 구현 범위를 정한 결과라고 설명한다.
- 코드 도움을 받은 부분도 내가 직접 흐름과 이유를 설명할 수 있는 파일을 중심으로 준비한다.


fragfarm-mobile 최종 정리 상태

- 홈, 상품 탐색, 검색, 상세, 장바구니, 위시리스트, 주문, 회원과 게시판까지 제출용 페이지 볼륨을 확보했다.
- 상품 카드의 정보 정렬, 상태 뱃지, 리뷰 아카이브 문구와 홈 모션을 브랜드 톤에 맞게 정리했다.
- 상품은 페이지당 10개로 조정하고 영문/한글 유사어 검색을 적용했다.
- 스킵 링크, 카테고리 포커스 복원, 카드 탭 순서, 다이얼로그 포커스와 reduced-motion 대응을 보완했다.
- 불필요한 console.log를 제거하고 전체 PHP/JavaScript 문법 검사와 주요 페이지 HTTP 응답 확인을 완료했다.
- fragfarm-mobile은 제출용 포트폴리오로 확정하고 다음 프로젝트 정리로 넘어간다.
